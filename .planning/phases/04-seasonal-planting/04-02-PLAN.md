**A. Create ui/src/pages/Calendar.jsx** — monthly planting calendar page (per D-05, D-06, D-07, D-08, D-14).

The page:
1. Calls getMe() on mount to get user's zoneId (auth check — redirect to /login on 401, same pattern as Dashboard)
2. If user has no zone set, shows a message: "Select your hardiness zone on the Dashboard to view your planting calendar." with a link to /dashboard
3. If user has a zone, calls getPlantingCalendar(zoneId) to load data
4. Renders MUI Tabs — one tab per month (January through December)
5. Highlights the current month tab (per D-08) by defaulting the active tab to the current month using `new Date().getMonth()`
6. Below the tabs, renders a MUI Table showing plant entries active in the selected month
7. Each table row shows: Plant Name, Plant Type, Method, Notes (per D-07)
8. A planting season entry is "active" in a month if the month index is between startMonth and endMonth (inclusive)

Key implementation:
```jsx
const MONTHS = [
  'January','February','March','April','May','June',
  'July','August','September','October','November','December'
];

function getActiveEntries(calendarData, month) {
  const monthIndex = MONTHS.indexOf(month);
  return calendarData.filter(entry => {
    const start = MONTHS.indexOf(entry.startMonth);
    const end = MONTHS.indexOf(entry.endMonth);
    return monthIndex >= start && monthIndex <= end;
  });
}
```

Use these MUI components:
```jsx

         TableContainer, TableHead, TableRow, Paper, CircularProgress,
         Alert, Box } from "@mui/material";

```

No special visual highlight beyond MUI's built-in selected tab indicator is needed — the current month IS the initially selected tab.
