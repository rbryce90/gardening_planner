    Create `server/scripts/seed.ts` — a standalone Deno script that:

    1. Imports:
       ```typescript
       import { DatabaseSync } from "node:sqlite";
       import seedData from "./seed-data.json" with { type: "json" };
       ```

    2. Opens the database directly (not using getDatabase() — this is a standalone script):
       ```typescript
       const db = new DatabaseSync("plants.db");
       db.exec("PRAGMA foreign_keys = ON;");
       ```

    3. Runs CREATE TABLE IF NOT EXISTS for all 6 tables (copy the exact DDL from plantDb.ts) so the script works on a fresh database too.

    4. Inserts in FK-dependency order:
       a. **zones** — `INSERT OR IGNORE INTO zones (name, min_temperature, max_temperature) VALUES (?, ?, ?)` — UNIQUE on `name` column provides idempotency.
       b. **plants** — `INSERT OR IGNORE INTO plants (name, category, growth_form, edible_part, family) VALUES (?, ?, ?, ?, ?)` — UNIQUE on `name` column provides idempotency.
       c. **plant_types** — Resolve `plant_name` to `plant_id` by querying `SELECT id FROM plants WHERE name = ?`. Then `INSERT OR IGNORE INTO plant_types (plant_id, name, scientific_name, description, planting_notes) VALUES (?, ?, ?, ?, ?)`. Add a UNIQUE constraint check: before inserting, the script should use `INSERT OR IGNORE` — but `plant_types` has no UNIQUE constraint in the schema. Use `INSERT INTO plant_types ... SELECT ?, ?, ?, ?, ? WHERE NOT EXISTS (SELECT 1 FROM plant_types WHERE plant_id = ? AND name = ?)` for idempotency.
       d. **companions** — Resolve `plant_name` and `companion_name` to plant IDs. Ensure lower ID is `plant_id`, higher is `companion_id`. Use `INSERT OR IGNORE` — but companions has no UNIQUE constraint either. Use the same NOT EXISTS pattern: `INSERT INTO companions (plant_id, companion_id) SELECT ?, ? WHERE NOT EXISTS (SELECT 1 FROM companions WHERE plant_id = ? AND companion_id = ?)`.
       e. **antagonists** — Same pattern as companions. Resolve names to IDs, enforce lower-first ordering, use NOT EXISTS for idempotency.
       f. **planting_seasons** — Resolve `plant_type_name` to `plant_type_id` (via `SELECT id FROM plant_types WHERE name = ?`) and `zone_name` to `zone_id` (via `SELECT id FROM zones WHERE name = ?`). Use NOT EXISTS: `INSERT INTO planting_seasons ... SELECT ?, ?, ?, ?, ?, ? WHERE NOT EXISTS (SELECT 1 FROM planting_seasons WHERE plant_type_id = ? AND zone_id = ?)`.

    5. Log counts after each entity type insertion:
       ```typescript
       console.log(`Inserted ${count} zones`);
       console.log(`Inserted ${count} plants`);
       // etc.
       ```

    6. Close the database: `db.close();`

    7. Print a final summary: `console.log("Seed complete.");`

    Important: The script runs from the `server/` directory (the deno.json task `cd`s there), so the DB path is just `"plants.db"`.
